# 一、协议格式

TCP是传输层协议。一个TCP由四元组组成【local ip, local port,remote ip,remote port】，其中一个不同都意味着新的TCP连接。TCP的特点是面向连接、可靠传输、流量控制。应用场景是用来做可靠的数据传输，例如HTTP协议、RPC协议等应用层协议是基于TCP进行的数据传输。

# 三、特点

TCP最大的特点就是提供可靠传输，以下几个机制就是保证了其可靠传输的特点。

# 3.1 可靠传输

超时重传机制

SYN包或者FIN包长时间没接收到ACK包后，会触发超时重传机制，每次重传间隔是上次的两倍。

重传次数可以通过配置操作系统的TCP相关的参数

ACK机制

TCP提供ACK包确认机制，通过对序列号+1进行确认，确保FIN，SYN包和其他数据传输包被正常接收。

# 3.2 流量控制

流量控制是对接收方接收能力的控制，根据接收方当先负载，调整发送端和接收端的窗口，达到稳定传输的目的。

发送流量 = min(拥塞窗口，滑动窗口)

滑动窗口

滑动窗口是流量控制的具体实现，分为接收窗口和发送窗口，发送窗口会根据接收窗口进行适用。举个例子：

首先双方经过三次握手建立连接后，初始化各自的窗口大小，均为200字节。此时发送100个TPC包，接收端接收到后由于当前的负载比较大，60个字节进入缓冲队列，那么接收端将窗口大小缩小为140，并且在ACK的时候将配置窗口大小为140，发送端在接收到ACK包后将窗口进行调整。

# 3.3 拥塞控制

拥塞控制是对发送方发送的控制，由于在开始传输的时候，TCP并不知道当前的网络情况，如果网络环境很差特别容易丢失，那么就需要控制发送端的发送速度，否则会造成大量丢包重试。

拥塞窗口

拥塞窗口由四个算法控制

  • 慢启动算法：通过从0开始指数增长的方式扩大拥塞窗口，每经过一个RTT窗口进行翻倍，当到达慢启动阈值后（16），采用拥塞避免算法增长。

    • 拥塞避免算法：每经过一个RTT，窗口增加1

  • 拥塞发生算法：当发现三次重复包的ACK后，说明当前出现了包的丢失，故进行快重传。

  • 快速恢复算法：当发现三次重复包的ACK后，说明当前网络拥塞，将拥塞阈值和窗口大小调为当前窗口的一半，启动快恢复算法（线程增长）

# 四、应用调优

Socket 编程时，单机最多可以建立多少个 TCP 连接，受到操作系统的影响。Windows 下单机的TCP连接数受多个参数影响：

  • 最大TCP连接数
  • 最大动态端口数
  • 最大TCB数量
  • 最大TCB Hash table数量
