# 面向对象

Go语言

  • 没有封装的特性，所有的方法和属性都是分散的，但是可以进行绑定

  • 有继承的特性，可以以匿名struct的方式继承其他struct及其method

  • 多态的特性，父类struct可以接收子类struct的值，并且通过父类或接口调用子类（继承父类重写）或（实现接口）的方法。

method定义

除了上述的函数之外，Go还可以创建自定义变量的方法，结构体内部不能拥有方法，方法通过以下格式与结构体绑定。这样相同方法名不同参数的写法类似于Java中的方法重载。

// funcName方法属于ReceiverType结构体，可以通过结构体.funcName调用该方法
func (r ReceiverType) funcName(parameters) (results)

type Rectangle struct {
    width, height float64
}

func (r Rectangle) area() float64 {
    return r.width*r.height
}

func main() {
    r1 := Rectangle{12, 2}

    fmt.Println("Area of r1 is: ", r1.area())
}

# interface

interface是一组 method 签名的组合，我们通过 interface 来定义对象的一组行为。如果某个对象实现了某个接口的所有方法，则此对象就实现了此接口。

  • interface多态：interface变量可以存放（赋值）任意类型实现了该interface的对象，类似Java中的多态
  • 空 interface： 可以存储任意类型的数值，方法返回interface{}，说明该方法可以返回任意值
// 定义 a 为空接口
var a interface{}
var i int = 5
s := "Hello world"
// a 可以存储任意类型的数值
a = i
a = s

# 并发

  • 协程的开销 (opens new window)
  • 进程和线程上下文切换的开销 (opens new window)

Go语言支持并发编程，goroutine 是 Go 并行设计的核心。goroutine本质上就是协程，与进程和线程的关系类似，线程和协程师一对多的关系，协程共享线程的数据。特点是：

  • 内存占用小：协程为2KB左右，而线程要几M，在大并发下创建线程会导致内存被大量占用
  • 上下文切换开销小：共享线程的大部分数据，只切换协程小部分的数据
  • 不需要做并发处理：在同个线程内，所以不需要做加锁和其他并发处理（可见性），协程间通过管道通信
  • 需要手动控制调度

Go语言中通过go关键字创建协程，runtime 包中有几个处理 goroutine 的函数：

  • Gosched()：让出CPU给其他协程
  • Goexit：退出当前执行的 goroutine，但是 defer 函数还会继续调用
  • NumGoroutine：返回正在执行和排队的任务总数
  • NumCPU：返回 CPU 核数量
// 在本案例中，由于协程本身就是运行在单线程中，所以go创建的协程不一定会被执行，很可能还没执行

func say(s string) {
	for i := 0; i < 5; i++ {
    // 表示让 CPU 把时间片让给别人，下次某个时候继续恢复执行该 goroutine
		runtime.Gosched()
		fmt.Println(s)
	}
}


func main() {
	go say("world") // 开一个新的 Goroutines 执行
	say("hello") // 当前 Goroutines 执行
}

goroutine间通过channel进行通信，channel是一个通信管道，提供了双向通信功能，可以通过他发送和接收数据。默认的channel的大小为0，每次发送/接收数据都会阻塞直到数据被接收/发送。

ci := make(chan int)
cs := make(chan string)
cf := make(chan interface{})

v := <-ch  // 从 ch 中接收数据，并赋值给v

select

select可以实现多路复用的功能，通过select可以监听多个channel，select默认是阻塞的，监听到消息后会随机选择一个管道执行。

// 此处的变量声明采用省略式写法，等价于：（c chan int, quit chan int）
func fibonacci(c, quit chan int) {
	x, y := 1, 1
	for {
		select {
    // 向管道c发送元素，并阻塞直到有人接收则触发代码
		case c <- x:
      // 等价于x=y，y=x+y
			x, y = y, x + y 
    // 监听管道quit并阻塞，直到收到管道消息并触发代码
		case <-quit:
			fmt.Println("quit")
			return
		}
	}
}

func main() {
	c := make(chan int)
	quit := make(chan int)
	go func() {
		for i := 0; i < 10; i++ {
      // 匿名式写法接收数据
			fmt.Println(<-c)
		}
		quit <- 0
	}()
	fibonacci(c, quit)
}

# 三、Go常见面试题

通过面试题快速学习Go语言

# 个人Go体验

  • 代码简洁灵活
    • 类的公私有：首字母大小写
    • 变量定义：可以不指定具体类型
    • 方法返回值：多值返回
    • 继承非常方便：直接将某个struct结构体作为字段即可拥有其所有字段，通过类似struct.field.field的方式读取变量
  • 可读性较差
    • 自定义method：Java是聚合到某个类中，有哪些方法一目了然，而Go可能分散在各个地方，可读性较差
    • 灵活带来的可能的可读性差：比如返回值可以匿名多返回，而不声明返回类型及变量名可能导致可读性降低
  • 函数编程让人头晕

# Go和Java有什么不同

  • 参考一 (opens new window)
  1. 性能上：golang的性能比Java更好，占用内存更少，使用goroutine避免内核态和用户态切换成本
  2. 编译部署：Java通过虚拟机编译，使用JVM跨平台编译；Go中不存在虚拟机，而是针对不同的平台，编译对应的机器码
  3. 功能上：
    • 接口：java等面向对象编程的接口是侵入式接口，需要明确声明自己实现了某个接口。 而Golang的非侵入式接口不需要通过任何关键字，只要一个类型实现了接口的所有方法，就是这个接口的实现。
    • 继承：Java的继承通过extends关键字完成，不支持多继承；Go语言的继承通过Struct的方式，子类只需要把基类作为成员放在子类的定义中，支持多继承。
    • 异常处理：java中错误（Error）和异常(Exception)被分类管理，golang中只有error，一旦发生错误逐层返回，直到被处理。
    • 并发：通常借助于共享内存（全局变量）作为线程间通信的媒介，通常会有线程不安全问题，使用了加锁（同步化）、使用原子类、使用volatile提升可见性等解决；但在Golang中使用的是通道（channel）作为协程间通信的媒介，多个goroutine之间通过Channel来通信，chan的读取和写入操作为原子操作，所以是安全的。
